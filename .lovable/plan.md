## Problem

In Jose Luis Cespedes' **Aviso de Despido**, the cédula prints as `***-*******-X` instead of the full ID. Same issue would affect every HR letter (Carta de Trabajo, Constancia, Carta de Recomendación, Liquidación, etc.) generated by anyone who isn't `admin` or `management`.

## Root cause

`EmployeeDetailDialog` (and by extension `EmployeeLetterDialog`) reads the employee from the **`employees_safe` view**, which masks PII:

```sql
CASE WHEN has_role(auth.uid(), 'admin') OR has_role(auth.uid(), 'management')
     THEN cedula
     ELSE '***-*******-' || right(cedula, 1)
END AS cedula
```

The current user has the `office` role, so the view returns the masked string. That masked string is then:
1. Displayed in the dialog header (acceptable — directory PII protection), AND
2. **Passed through to the generated PDF letter** (broken — legal HR documents require the full cédula).

The mask was designed to protect the directory listing from casual viewing, but it's leaking into authoritative HR documents.

## Fix

The `office` role exists specifically to handle HR paperwork — they need the real cédula to issue legally valid letters. Two coordinated changes:

### 1. Update the `employees_safe` view

Add `office` to the unmasked branch for `cedula` only. Keep `bank_account_number` masked for `office` (they don't need bank info for letters).

```sql
CREATE OR REPLACE VIEW public.employees_safe
WITH (security_invoker = on) AS
SELECT
  id, name, position, date_of_hire, date_of_termination, salary,
  is_active, shirt_size, pant_size, boot_size, date_of_birth,
  entity_id, sex, created_at, updated_at,
  CASE
    WHEN has_role(auth.uid(), 'admin'::app_role)
      OR has_role(auth.uid(), 'management'::app_role)
      OR has_role(auth.uid(), 'office'::app_role)
    THEN cedula
    ELSE '***-*******-' || right(cedula, 1)
  END AS cedula,
  bank,
  CASE
    WHEN has_role(auth.uid(), 'admin'::app_role)
      OR has_role(auth.uid(), 'management'::app_role)
    THEN bank_account_number
    ELSE '****' || right(COALESCE(bank_account_number, ''), 4)
  END AS bank_account_number
FROM employees;
```

Note: I'll also clean up the dead `CASE` on `bank` (currently both branches return `bank`) — just select `bank` directly.

### 2. No React changes needed

Once the view returns the unmasked cédula for `office` users, the dialog and the generated PDF will pick it up automatically.

## Out of scope

- `accountant`, `supervisor`, `viewer`, `driver` stay masked — they don't issue HR letters.
- `bank_account_number` masking for `office` is unchanged.
- No changes to `employees` base table RLS.
